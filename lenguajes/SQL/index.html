¿Qué es SQL?

SQL (Structured Query Language) es el lenguaje estándar para gestionar bases de datos relacionales. Te permite crear, leer, actualizar y eliminar datos de forma eficiente.

Características principales:

  • Relacional: Datos organizados en tablas con relaciones
  • ACID: Garantiza integridad y consistencia de datos
  • Escalable: Maneja millones de registros eficientemente
  • Estándar: Usado en MySQL, PostgreSQL, SQL Server, Oracle
  • Potente: Consultas complejas con JOINs y agregaciones

¿Para qué sirve?

SQL es esencial para:

  • Gestionar datos de aplicaciones web y móviles
  • Análisis de datos y reportes
  • Data warehousing y business intelligence
  • Transacciones bancarias y comercio electrónico
  • Cualquier sistema que almacene información

Ventajas y desventajas

✅ Ventajas ❌ Limitaciones
Consultas potentes y flexibles Requiere diseño previo (esquema)
Seguridad y control de acceso Menos flexible que NoSQL para datos no estructurados
Integridad referencial garantizada Curva de aprendizaje para consultas complejas
Transacciones ACID Escalabilidad horizontal más compleja

Fundamentos de SQL

1️⃣ SELECT - Seleccionar datos

La consulta más fundamental. Recupera datos de una tabla.

SELECT * FROM usuarios;
SELECT nombre, email FROM usuarios;
SELECT nombre FROM usuarios WHERE edad > 18;

2️⃣ INSERT - Insertar datos

Añade nuevos registros a una tabla.

INSERT INTO usuarios (nombre, email, edad) 
VALUES ('Juan', 'juan@email.com', 25);

INSERT INTO usuarios VALUES (1, 'María', 'maria@email.com', 30);

3️⃣ UPDATE - Actualizar datos

Modifica registros existentes.

UPDATE usuarios SET edad = 26 WHERE nombre = 'Juan';
UPDATE usuarios SET email = 'nuevo@email.com' WHERE id = 1;

4️⃣ DELETE - Eliminar datos

Borra registros de una tabla.

DELETE FROM usuarios WHERE id = 1;
DELETE FROM usuarios WHERE edad < 18;

5️⃣ WHERE - Filtros

Especifica condiciones para las consultas.

WHERE edad > 18
WHERE nombre = 'Juan' AND edad > 25
WHERE ciudad IN ('Madrid', 'Barcelona')
WHERE email LIKE '%@gmail.com'

6️⃣ ORDER BY - Ordenamiento

Ordena los resultados ascendente o descendentemente.

SELECT * FROM usuarios ORDER BY edad ASC;
SELECT * FROM usuarios ORDER BY nombre DESC;
SELECT * FROM usuarios ORDER BY edad DESC, nombre ASC;

7️⃣ LIMIT - Limitar resultados

Restringe la cantidad de registros retornados.

SELECT * FROM usuarios LIMIT 10;
SELECT * FROM usuarios LIMIT 5 OFFSET 10;
SELECT * FROM usuarios ORDER BY edad DESC LIMIT 3;

Conceptos Intermedios

1️⃣ JOIN - Relacionar tablas

Combina datos de múltiples tablas basado en relaciones.

SELECT u.nombre, p.titulo 
FROM usuarios u 
INNER JOIN posts p ON u.id = p.usuario_id;

SELECT u.nombre, p.titulo 
FROM usuarios u 
LEFT JOIN posts p ON u.id = p.usuario_id;

2️⃣ COUNT - Contar registros

Cuenta filas que cumplen una condición.

SELECT COUNT(*) FROM usuarios;
SELECT COUNT(*) as total FROM usuarios WHERE edad > 18;
SELECT nombre, COUNT(*) FROM posts GROUP BY usuario_id;

3️⃣ SUM, AVG, MIN, MAX - Agregaciones

Calcula valores agregados de columnas numéricas.

SELECT SUM(salario) FROM empleados;
SELECT AVG(edad) FROM usuarios;
SELECT MIN(fecha), MAX(fecha) FROM pedidos;

4️⃣ GROUP BY - Agrupar datos

Agrupa filas con valores idénticos en una columna.

SELECT departamento, COUNT(*) as total 
FROM empleados 
GROUP BY departamento;

SELECT usuario_id, COUNT(*) as posts 
FROM articulos 
GROUP BY usuario_id 
HAVING COUNT(*) > 5;

5️⃣ DISTINCT - Valores únicos

Muestra valores únicos sin repeticiones.

SELECT DISTINCT ciudad FROM usuarios;
SELECT DISTINCT departamento FROM empleados;
SELECT COUNT(DISTINCT usuario_id) FROM pedidos;

6️⃣ LIKE - Búsqueda de patrones

Busca valores que coincidan con un patrón.

SELECT * FROM usuarios WHERE nombre LIKE 'J%';
SELECT * FROM usuarios WHERE email LIKE '%gmail.com';
SELECT * FROM usuarios WHERE nombre LIKE '%ar%';

7️⃣ IN y BETWEEN - Rangos y listas

Filtra valores dentro de listas o rangos.

SELECT * FROM usuarios WHERE edad IN (18, 25, 30);
SELECT * FROM usuarios WHERE edad BETWEEN 20 AND 30;
SELECT * FROM pedidos WHERE fecha BETWEEN '2024-01-01' AND '2024-12-31';

Conceptos Avanzados

1️⃣ Subqueries - Consultas anidadas

Consultas dentro de otras consultas para lógica compleja.

SELECT nombre FROM usuarios 
WHERE id IN (SELECT usuario_id FROM posts WHERE likes > 100);

SELECT AVG(salario) FROM empleados 
WHERE salario > (SELECT AVG(salario) FROM empleados);

2️⃣ UNION - Combinar resultados

Combina resultados de múltiples consultas SELECT.

SELECT nombre, 'usuario' as tipo FROM usuarios
UNION
SELECT nombre, 'admin' as tipo FROM administradores;

SELECT email FROM usuarios
UNION ALL
SELECT email FROM clientes;

3️⃣ CASE - Lógica condicional

Implementa condiciones if-else en consultas.

SELECT nombre, 
  CASE 
    WHEN edad < 18 THEN 'Menor'
    WHEN edad < 65 THEN 'Adulto'
    ELSE 'Jubilado'
  END as categoria
FROM usuarios;

SELECT nombre,
  CASE edad WHEN 25 THEN 'Especial' ELSE 'Normal' END
FROM usuarios;

4️⃣ Window Functions - Funciones de ventana

Análisis avanzado con funciones sobre particiones.

SELECT nombre, salario,
  ROW_NUMBER() OVER (ORDER BY salario DESC) as rango
FROM empleados;

SELECT nombre, salario, departamento,
  AVG(salario) OVER (PARTITION BY departamento) as promedio_dept
FROM empleados;

5️⃣ CTE - Expresiones de tabla común

Define consultas reutilizables dentro de una consulta principal.

WITH salarios_altos AS (
  SELECT * FROM empleados WHERE salario > 50000
)
SELECT departamento, COUNT(*) 
FROM salarios_altos 
GROUP BY departamento;

WITH RECURSIVE numeros AS (
  SELECT 1 as n
  UNION ALL
  SELECT n+1 FROM numeros WHERE n < 10
)
SELECT * FROM numeros;

6️⃣ Transacciones - BEGIN, COMMIT, ROLLBACK

Garantiza atomicidad en múltiples operaciones.

BEGIN TRANSACTION;
  UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1;
  UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;

BEGIN TRANSACTION;
  DELETE FROM usuarios WHERE id = 5;
ROLLBACK;

7️⃣ Índices y Optimización

Mejora el rendimiento de las consultas.

CREATE INDEX idx_email ON usuarios(email);
CREATE INDEX idx_user_date ON posts(usuario_id, fecha);
DROP INDEX idx_email;

EXPLAIN SELECT * FROM usuarios WHERE email = 'test@email.com';

Ejemplos Complejos Prácticos

1. Sistema de Tienda Online

Objetivo: Gestionar clientes, productos, pedidos y detalles de pedidos.

-- Crear tablas
CREATE TABLE clientes (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    ciudad VARCHAR(50),
    fecha_registro DATE
);

CREATE TABLE productos (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100),
    precio DECIMAL(10,2),
    stock INT,
    categoria VARCHAR(50)
);

CREATE TABLE pedidos (
    id INT PRIMARY KEY AUTO_INCREMENT,
    cliente_id INT NOT NULL,
    fecha DATE,
    total DECIMAL(10,2),
    FOREIGN KEY (cliente_id) REFERENCES clientes(id)
);

CREATE TABLE detalles_pedido (
    id INT PRIMARY KEY AUTO_INCREMENT,
    pedido_id INT NOT NULL,
    producto_id INT NOT NULL,
    cantidad INT,
    precio_unitario DECIMAL(10,2),
    FOREIGN KEY (pedido_id) REFERENCES pedidos(id),
    FOREIGN KEY (producto_id) REFERENCES productos(id)
);

-- Consulta: Top 5 clientes por gasto total
SELECT c.nombre, SUM(p.total) as gasto_total, COUNT(p.id) as num_pedidos
FROM clientes c
LEFT JOIN pedidos p ON c.id = p.cliente_id
GROUP BY c.id, c.nombre
ORDER BY gasto_total DESC
LIMIT 5;

2. Panel de Análisis de Redes Sociales

Objetivo: Analizar rendimiento de posts, usuarios activos y tendencias.

-- Tabla de usuarios y posts
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nombre VARCHAR(100),
    fecha_creacion DATE
);

CREATE TABLE posts (
    id INT PRIMARY KEY,
    usuario_id INT,
    contenido TEXT,
    likes INT DEFAULT 0,
    comentarios INT DEFAULT 0,
    fecha DATE,
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id)
);

-- Consulta: Usuarios más activos del mes actual
SELECT u.nombre, COUNT(p.id) as num_posts, SUM(p.likes) as total_likes
FROM usuarios u
INNER JOIN posts p ON u.id = p.usuario_id
WHERE MONTH(p.fecha) = MONTH(CURDATE()) 
  AND YEAR(p.fecha) = YEAR(CURDATE())
GROUP BY u.id
ORDER BY total_likes DESC;

-- Posts virales (más de 1000 likes)
SELECT u.nombre, p.contenido, p.likes, p.fecha
FROM posts p
INNER JOIN usuarios u ON p.usuario_id = u.id
WHERE p.likes > 1000
ORDER BY p.likes DESC;

-- Engagement rate por usuario
SELECT u.nombre,
  COUNT(p.id) as posts,
  ROUND(AVG(p.likes), 2) as prom_likes,
  ROUND(AVG(p.comentarios), 2) as prom_comentarios
FROM usuarios u
LEFT JOIN posts p ON u.id = p.usuario_id
GROUP BY u.id
HAVING COUNT(p.id) > 0;

3. Gestión de Nóminas y Departamentos

Objetivo: Calcular salarios, comisiones y análisis departamental.

-- Tablas de empleados y departamentos
CREATE TABLE departamentos (
    id INT PRIMARY KEY,
    nombre VARCHAR(100),
    presupuesto DECIMAL(12,2)
);

CREATE TABLE empleados (
    id INT PRIMARY KEY,
    nombre VARCHAR(100),
    departamento_id INT,
    salario_base DECIMAL(10,2),
    comisión DECIMAL(5,2) DEFAULT 0,
    fecha_contratacion DATE,
    FOREIGN KEY (departamento_id) REFERENCES departamentos(id)
);

-- Salario total con comisión (Window function)
SELECT nombre, departamento_id,
  salario_base,
  ROUND(salario_base * comisión / 100, 2) as comisión_monto,
  ROUND(salario_base * (1 + comisión/100), 2) as salario_total,
  ROW_NUMBER() OVER (PARTITION BY departamento_id ORDER BY salario_base DESC) as rango_dept
FROM empleados;

-- Comparación con promedio departamental
SELECT e.nombre, d.nombre as departamento, e.salario_base,
  ROUND(AVG(e2.salario_base) OVER (PARTITION BY e.departamento_id), 2) as prom_depto,
  CASE 
    WHEN e.salario_base > AVG(e2.salario_base) OVER (PARTITION BY e.departamento_id) 
      THEN 'Arriba del promedio'
    ELSE 'Bajo el promedio'
  END as categoria
FROM empleados e
INNER JOIN departamentos d ON e.departamento_id = d.id
CROSS JOIN empleados e2;
